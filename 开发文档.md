# 中国象棋机械臂对弈系统 - 开发文档

## 1. 项目概述

### 1.1 项目目标
实现一个能够自主下中国象棋的机械臂系统，包括棋盘检测、棋子识别、最佳走法获取和机械臂执行。

### 1.2 系统组成
- **硬件**: DOFBOT 6轴机械臂、树莓派5、USB相机
- **软件**: Python 3.x, OpenCV, YOLO, UCI引擎
- **通信**: TCP socket (端口5000)

## 2. 技术架构

### 2.1 客户端 (chess_arm_client.py)
运行在电脑上，负责：
1. 读取相机画面检测棋盘
2. 生成FEN格式字符串
3. 调用引擎获取最佳走法
4. 发送MOVE指令给服务器
5. 显示棋盘和走法供用户确认

### 2.2 服务器 (chess_arm_server.py)
运行在树莓派上，负责：
1. 接收MOVE指令
2. 执行IK求解
3. 控制机械臂走棋
4. 控制气泵吸取/放下棋子

### 2.3 坐标系统
- 棋盘坐标: 9列(a-i) x 10行(0-9)
- 机器人坐标: [x, y, z] 单位cm
- 转换公式: `board_to_robot.py` 中的线性映射

## 3. 核心模块

### 3.1 棋盘检测 (board_detector.py)
- 使用OpenCV检测棋盘角点
- 校正相机畸变
- 输出8x8交叉点网格

### 3.2 棋子检测 (piece_detector.py)
- 使用YOLO模型检测棋子
- 生成标准FEN格式
- 支持红/黑双方棋子识别

### 3.3 坐标转换 (coord_utils.py)
```
UCCI格式 <-> 内部数字坐标
a9 -> (0, 0)
i0 -> (8, 9)
```

### 3.4 逆运动学 (ik.py)
- 输入: 机器人坐标 (x, y, z)
- 输出: 6个舵机角度
- 联动长度: P=5, A1=12.1, A2=8.4, A3=12.6, A4=18.1

### 3.5 本地引擎 (simple_engine.py)
- 使用Pikafish引擎
- 支持中国象棋FEN格式
- 返回最佳走法(UCCI格式)

## 4. 走棋流程详解

### 4.1 完整流程
```
1. 客户端读取相机画面
2. 检测棋盘角点，校正畸变
3. 检测棋子位置，生成FEN
4. 调用simple_engine获取最佳走法
5. 坐标转换 (UCCI -> robot)
6. 发送MOVE命令到服务器
7. 服务器执行走棋序列
```

### 4.2 机械臂执行序列
```
初始位置
    ↓
起始位置上方 (safe_height=15cm)
    ↓
一次下降 (8cm)
    ↓
二次下降 (piece_height=5cm)
    ↓
气泵吸取棋子
    ↓
抬升到安全高度
    ↓
移动到目标位置上方
    ↓
一次下降 (8cm)
    ↓
二次下降 (piece_height=5cm)
    ↓
气泵放下棋子
    ↓
抬升到安全高度
    ↓
返回初始位置
```

## 5. 关键技术点

### 5.1 Z轴校正
公式: `z_corrected = z + k * sqrt(x^2 + y^2)`
- k值通过实际测试调整
- 用于补偿机械臂在不同位置的高度误差

### 5.2 两步下降
- 避免直接从高处下降导致棋子移位
- 第一次下降到安全高度
- 第二次下降到棋子高度

### 5.3 吃子处理
- 通过FEN判断目标位置是否有棋子
- 先移除被吃棋子到丢弃位置
- 再执行正常走棋

### 5.4 黑方视角处理
- 引擎返回的是红方视角
- 需要rotate_move_180转换
- 吃子判定使用原始坐标

## 8. 已知问题与解决方案

### 8.1 坐标方向错误
- 现象: 黑方走棋位置错误
- 解决: 确认rotate_move_180是否正确处理

### 8.2 吃子判定错误
- 现象: 自己吃自己或吃空气
- 解决: 使用原始坐标进行吃子判定，不经过旋转

### 8.3 Z轴高度不一致
- 现象: 不同位置高度不同
- 解决: 调整z_correction_factor参数
